## 🔥 必读

### 1. vLLM: Efficient Memory Management for Large Language Model Serving with PagedAttention

- **作者**: Woosuk Kwon, Zhuohan Li, Siyuan Zhuang, Ying Sheng, Lianmin Zheng, Cody Hao Yu, Joseph E. Gonzalez, Hao Zhang, Ion Stoica
- **机构**: UC Berkeley
- **链接**: [arXiv](https://arxiv.org/abs/2309.06180) | [PDF](https://arxiv.org/pdf/2309.06180)
- **来源**: 🔥 HF Trending，⬆️ 49
- 📒 **笔记**: [[vLLM]]

- **核心方法**: 提出 [[PagedAttention]] 算法，将 KV cache 内存管理类比操作系统虚拟内存分页机制。vLLM 通过按需分配、灵活共享 KV cache block，实现接近零浪费的内存利用率。吞吐量提升 2-4x，延迟不变。
- **对比方法**: 对比 [[FasterTransformer]]、[[Orca]] 等现有 serving 系统。
- **借鉴意义**: 如果你在做 domain-specific LLM 的部署落地，vLLM 是当前最优选。PagedAttention 的设计思路对任何需要动态内存管理的推理系统都有参考价值。
- **锐评**: 这篇已经是事实上的 LLM serving 标准了。49 upvotes 不是偶然。方法简洁、工程实现扎实、开源可用。唯一的"硬伤"是论文本身写得偏工程报告，理论分析较浅。但实用性压倒一切。🔥

### 2. Simula: Reasoning-Driven Synthetic Data Generation and Evaluation

- **作者**: Tim R. Davidson, Benoit Seguin, Enrico Bacis, Cesar Ilharco, Hamza Harkous
- **机构**: 未知（摘要未明确）
- **链接**: [arXiv](http://arxiv.org/abs/2603.29791v1) | [PDF](https://arxiv.org/pdf/2603.29791v1)
- **来源**: 📄 arXiv 关键词检索
- 📒 **笔记**: [[Simula]]

![](https://arxiv.org/html/2603.29791v1/x1.png)

- **核心方法**: 提出 **seedless agentic** 框架——不需要 seed data，通过推理驱动的 agent 自主生成 synthetic dataset。用户可以定义 dataset 特性（规模、分布、难度），系统通过 explainable process 控制生成过程。
- **对比方法**: 对比依赖 seed data 或 manual prompt 的现有方法。
- **借鉴意义**: 对做 instruction tuning data、domain-specific training data 的人非常有价值。无需 seed 意味着可以快速填补数据空白，尤其适合 vertical domain。
- **锐评**: 思路有意思。摆脱 seed 依赖是 synthetic data 的下一阶段挑战。但论文的"推理驱动"概念略显抽象，实验覆盖面还需要更多验证。能否真正替代 human annotation？目前只能说"有潜力"。🔥

## 👀 值得看

### 3. ASI-Evolve: AI Accelerates AI

- **作者**: Weixian Xu, Tiantian Mi, Yixiu Liu, Yang Nan, Zhimeng Zhou, Lyumanshan Ye, Lin Zhang, Yu Qiao, Pengfei Liu
- **机构**: 未知
- **链接**: [arXiv](http://arxiv.org/abs/2603.29640v1) | [PDF](https://arxiv.org/pdf/2603.29640v1)
- **来源**: 📄 arXiv 关键词检索

![](http://arxiv.org/html/2603.29640v1/figures/architecture/asi-evolve.png)

- **核心方法**: AI-for-AI 框架，通过 learn-design-experiment-analyze 循环让 AI 自主发现更好的 **数据、架构、学习算法**。在 pretraining data curation 上，evolved pipeline 提升 MMLU 18+ 分。
- **对比方法**: 对比 [[GRPO]]、[[DeltaNet]] 等方法。
- **借鉴意义**: "AI 设计 AI 训练数据"的思路对 data curation 工作有启发。自动化 pipeline 可以减少人工调参成本。
- **锐评**: 名字有点标题党，但内容有干货。三个方向（data/architecture/algorithm）都有实验验证。问题是：这是"自动化调参"还是真正的"发现"？方法论上的创新程度需要细看。👀

### 4. 6GAgentGym: Tool Use, Data Synthesis, and Agentic Learning for Network Management

- **作者**: Jiao Chen, Jianhua Tang, Xiaotong Yang, Zuohong Lv
- **机构**: 未知
- **链接**: [arXiv](http://arxiv.org/abs/2603.29656v1) | [PDF](https://arxiv.org/pdf/2603.29656v1)
- **来源**: 📄 arXiv 关键词检索

![](http://arxiv.org/html/2603.29656v1/x1.png)

- **核心方法**: 6G 网络管理场景的 agent 训练框架。通过 **Self-Instruct** 生成 closed-loop 训练轨迹，结合 SFT + RL。8B 模型达到 GPT-5 级别成功率。
- **对比方法**: 对比 static question benchmark 方法。
- **借鉴意义**: "Self-Instruct + execution verification" 的数据生成流程可以迁移到其他 domain。关键是用 Experiment Model 做验证，避免生成无效数据。
- **锐评**: 6G 场景太 niche，但数据生成方法论值得借鉴。closed-loop interaction + verification 的设计是关键。如果你在做 agent 训练数据，这个框架值得参考。👀

### 5. Amalgam: Hybrid LLM-PGM Synthesis Algorithm for Accuracy and Realism

- **作者**: 未明确
- **机构**: 未知
- **链接**: [arXiv](http://arxiv.org/abs/2603.27254v1) | [PDF](https://arxiv.org/pdf/2603.27254v1)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: 混合 [[LLM]] 和 [[PGM]]（Probabilistic Graphical Models）生成 synthetic dataset。LLM 保证 realism，PGM 保证可解释性和分析能力。
- **对比方法**: 对比 pure LLM 和 pure PGM 方法。
- **借鉴意义**: 对 healthcare、financial 等需要可解释 synthetic data 的 domain 有价值。LLM 生成的数据好看但难分析，PGM 反过来——混合两者是个务实思路。
- **锐评**: 思路务实。LLM 的 realism + PGM 的 structure，两边优点都保留。但 hybrid 方法通常意味着更复杂，工程落地成本高。如果你需要"既真实又能分析"的数据，可以考虑。👀

### 6. Baby Scale: Investigating Models Trained on Individual Children's Language Input

- **作者**: Steven Y. Feng, Alvin W. M. Tan, Michael C. Frank
- **机构**: 未明确
- **链接**: [arXiv](http://arxiv.org/abs/2603.29522v1) | [PDF](https://arxiv.org/pdf/2603.29522v1)
- **来源**: 📄 arXiv 关键词检索

![](http://arxiv.org/html/2603.29522v1/figures/scaling.png)

- **核心方法**: 用儿童语言输入数据（BabyView dataset）训练小规模 LM，研究 **human-scale 数据** 的学习效率。发现 grammar 任务 scaling 好，semantic/world knowledge 任务 scaling 差。
- **对比方法**: 对比 synthetic data 训练的模型。
- **借鉴意义**: 对"小数据高效学习"有启发。人类儿童用很少数据就能学会语言，LM 需要几个数量级更多——这个 gap 值得研究。
- **锐评**: 有趣的 cognitive science 视角。但对工程实践的直接指导有限。更多是"理解 data efficiency"的理论探索，而非给出 practical recipe。👀
